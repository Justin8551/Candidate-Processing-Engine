# Integrated Candidate Management System

A comprehensive C project developed for the Algorithm Design (PA) course at UPB ACS. The system manages a candidate recruitment process through multiple stages using advanced data structures: Queues, Binary Search Trees (BST), Heaps, and Graphs.

The project is divided into 4 main modules, each handling a specific stage of the recruitment and evaluation process.

Module 1: Candidate Management
- Implements a Queue to manage candidates in the order they apply.
- Provides functionalities to add, remove, and display candidates.

Module 2: Candidate Evaluation
- Utilizes a Binary Search Tree (BST) to store and manage candidate information based on their scores.
- Allows for efficient searching, insertion, and deletion of candidate records.

Module 3: Interview Scheduling
- Employs a Min-Heap to prioritize candidates based on their scores for interview scheduling.
- Facilitates the scheduling of interviews based on candidate priority.

Module 4: Candidate Interaction Graph
- Uses a Graph to represent interactions between candidates and interviewers.
- Supports functionalities to add interactions, display the graph, and analyze candidate-interviewer relationships.

The project is designed to be modular, allowing for easy maintenance and scalability.
Each module is implemented in separate files, and the main program integrates all modules to provide a seamless candidate management experience.
